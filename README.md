# Telegram Sari-Sari Sales Tracker Bot

A Telegram bot for sari-sari store owners that records daily sales in natural language, keeps a per-store price list, tracks stock, and answers revenue questions by day, week, or month.

## Features

- Natural-language sales logging with multiple items in one message
- Price list per store owner
- Optional stock and reorder tracking
- Daily, weekly, and monthly revenue summaries
- Top-selling items report
- Monthly scheduled sales summary

## Example Messages

- `3 coke 20, 2 lucky me 15, 1 soap 35`
- `sold 2 alaska 35 each and 1 bread 45`
- `sales summary today`
- `weekly revenue`
- `top selling items this month`
- `set price coke 20`
- `set stock coke 24 reorder 6`

## Supabase SQL

Run this in the Supabase SQL Editor:

```sql
create table if not exists public.sales_transactions (
  id bigint generated by default as identity primary key,
  telegram_user_id bigint not null,
  telegram_username text,
  total_amount numeric(12, 2) not null check (total_amount >= 0),
  currency text not null default 'PHP',
  raw_message text,
  created_at timestamptz not null default timezone('utc', now())
);

create index if not exists sales_transactions_user_idx
  on public.sales_transactions (telegram_user_id);

create index if not exists sales_transactions_created_at_idx
  on public.sales_transactions (created_at desc);

create table if not exists public.sales_line_items (
  id bigint generated by default as identity primary key,
  sales_transaction_id bigint not null references public.sales_transactions(id) on delete cascade,
  item_name text not null,
  quantity integer not null check (quantity > 0),
  unit_price numeric(12, 2) not null check (unit_price >= 0),
  line_total numeric(12, 2) not null check (line_total >= 0),
  created_at timestamptz not null default timezone('utc', now())
);

create index if not exists sales_line_items_transaction_idx
  on public.sales_line_items (sales_transaction_id);

create table if not exists public.price_list_items (
  id bigint generated by default as identity primary key,
  telegram_user_id bigint not null,
  item_name text not null,
  unit_price numeric(12, 2),
  stock_quantity integer,
  reorder_level integer,
  currency text not null default 'PHP',
  created_at timestamptz not null default timezone('utc', now()),
  updated_at timestamptz not null default timezone('utc', now()),
  unique (telegram_user_id, item_name)
);

create index if not exists price_list_items_user_idx
  on public.price_list_items (telegram_user_id);
```

## Environment Variables

- `TELEGRAM_BOT_TOKEN`
- `ANTHROPIC_API_KEY`
- `ANTHROPIC_MODEL`
- `SUPABASE_URL`
- `SUPABASE_KEY`
- `APP_BASE_URL`
- `TELEGRAM_WEBHOOK_SECRET`
- `TIMEZONE`
- `MONTHLY_SUMMARY_HOUR`
- `MONTHLY_SUMMARY_MINUTE`

Recommended values:

- `ANTHROPIC_MODEL=claude-haiku-4-5`
- `TIMEZONE=Asia/Manila`
- `MONTHLY_SUMMARY_HOUR=21`
- `MONTHLY_SUMMARY_MINUTE=0`

## Deploy to Railway

1. Push this project to GitHub.
2. Create a new Railway project from the repo.
3. Add the environment variables above.
4. Set `APP_BASE_URL` to the Railway public URL.
5. Let the included `Procfile` run:

```bash
uvicorn main:app --host 0.0.0.0 --port $PORT
```

6. Set the Telegram webhook:

```text
https://api.telegram.org/bot<TELEGRAM_BOT_TOKEN>/setWebhook?url=https://your-app.up.railway.app/webhook/<TELEGRAM_WEBHOOK_SECRET>
```
